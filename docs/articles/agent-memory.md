# Agent记忆机制设计

## 为什么需要记忆

智能体处理复杂任务时，仅依靠当前上下文无法持续理解用户目标。

记忆机制主要解决：

-   多轮任务连续性；
-   用户偏好保存；
-   历史任务恢复；
-   上下文长度控制。

------------------------------------------------------------------------

## 记忆不是保存全部历史

实际系统中，保存所有对话会导致：

-   Token增加；
-   无关信息干扰；
-   推理效率下降。

更合理的方式是保存有价值的信息：

-   当前任务；
-   用户目标；
-   已确认结论；
-   未完成事项。

------------------------------------------------------------------------

## 常见设计

短期记忆：

保存当前任务状态。

长期记忆：

保存长期有效的信息。

摘要记忆：

压缩历史内容，保留核心信息。

工程上需要根据业务选择合适方案。
