# 学习与面试笔记

## 1. 栏目定位

该栏目用于记录 AI 应用工程学习过程中形成的：

- 概念理解；
- 工程问题；
- 错误修正；
- 面试问题；
- 项目复盘；
- 技术取舍。

目标不是堆积零散笔记，而是把学习内容整理成可检索、可复用、可验证的知识体系。

---

## 2. 学习记录方法

推荐每个问题按以下结构整理：

```text
问题是什么
  ↓
为什么会出现
  ↓
核心原理
  ↓
最小示例
  ↓
常见误区
  ↓
项目中如何使用
  ↓
面试如何回答
```

这样可以避免只记住结论，却无法解释原因。

---

## 3. 每日问题模板

```markdown
# 问题标题

## 问题背景

在什么场景下遇到该问题。

## 核心结论

用简洁语言给出结论。

## 原理分析

解释为什么。

## 示例

给出代码、公式或业务案例。

## 常见误区

列出容易混淆的地方。

## 项目应用

说明在 Agent、RAG 或后端项目中如何使用。

## 面试回答

整理成 30 秒和 2 分钟两个版本。
```

---

## 4. 错误修正记录

学习过程中出现错误很正常，但需要保留修正过程。

建议记录：

- 原理解；
- 错误原因；
- 修正结论；
- 如何验证；
- 对项目设计的影响。

例如：

```text
原理解：
Recall@K 高说明检索系统整体没有问题。

修正：
Recall@K 高只能说明正确证据被召回。
如果 MRR 低，仍可能存在排序问题。

影响：
需要单独评估召回和重排，不应只看一个指标。
```

## 5. 面试题整理方式

面试题不建议只背定义。

每道题至少准备四层：

### 第一层：一句话定义

用于快速回答。

### 第二层：工作原理

说明内部流程。

### 第三层：项目应用

结合真实项目说明为什么使用。

### 第四层：问题与取舍

说明局限、异常和替代方案。

示例：

```text
问题：为什么使用 LangGraph？

定义：
LangGraph 是用于构建状态化 Agent 工作流的图编排框架。

原理：
通过 State、Node 和 Conditional Edge 管理流程与状态。

项目：
用于工业故障诊断中的阶段推进、中断恢复和人工审核。

取舍：
比普通 Chain 更可控，但需要自行设计状态和流程，开发成本更高。
```

## 6. 项目复盘模板

```markdown
# 项目复盘

## 业务目标

项目解决什么业务问题。

## 原始方案

最初如何设计。

## 实际问题

开发或测试中出现了什么问题。

## 根因分析

问题来自数据、检索、模型、流程还是工程架构。

## 调整方案

做了哪些改动。

## 评测结果

使用什么指标验证。

## 后续优化

仍有哪些未完成项。
```

## 11. 学习内容公开原则

博客公开前需要检查：

- 是否包含公司和客户隐私；
- 是否包含真实密钥、地址和内部数据；
- 是否把设计方案写成已上线能力；
- 技术结论是否经过验证；
- 是否存在无法解释的夸大指标；
- 图片是否仍引用本地绝对路径；
- 代码是否可以被误用于生产。

---

## 12. 总结

学习笔记的价值不在数量，而在于：

- 能否形成体系；
- 能否解释原理；
- 能否连接项目；
- 能否修正错误；
- 能否在面试中清晰表达。

持续记录问题、验证结论并完成项目复盘，才能把零散知识转化为稳定的工程能力。
