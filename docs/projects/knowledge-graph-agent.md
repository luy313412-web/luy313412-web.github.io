# 知识图谱招投标资格审查与项目匹配 Agent

## 项目背景

该项目面向企业投标前分析场景，通过知识图谱、大模型和智能流程编排能力，辅助企业判断项目匹配程度、资格符合情况以及潜在风险。

传统投标审核过程中，招标文件、企业资质、人员证书、历史业绩等信息分散，人工审核存在效率低、容易遗漏和难以快速分析的问题。

------------------------------------------------------------------------

## 业务目标

系统支持：

-   招标条件解析；
-   企业资质匹配；
-   人员证书检查；
-   历史业绩分析；
-   缺失条件识别；
-   风险提示。

输出项目匹配结果、风险说明以及补充建议。

------------------------------------------------------------------------

## 系统流程

``` text
招标文件采集
 ↓
文档解析
 ↓
大模型信息抽取
 ↓
知识图谱构建
 ↓
企业能力匹配
 ↓
查询分析
 ↓
生成辅助报告
```

------------------------------------------------------------------------

## 知识图谱设计

核心实体：

-   企业；
-   招标项目；
-   资质；
-   人员；
-   证书；
-   历史业绩。

核心关系：

-   企业拥有资质；
-   人员持有证书；
-   企业参与项目；
-   项目要求对应能力。

通过结构化关系支持复杂条件分析。

------------------------------------------------------------------------

## 大模型信息抽取

利用大模型处理非结构化招标文件：

-   识别资格条件；
-   提取关键实体；
-   生成结构化信息；
-   构建图谱数据。

通过结构化输出约束，提高抽取结果稳定性。

------------------------------------------------------------------------

## 图谱查询

技术方案：

-   Neo4j 存储关系数据；
-   Text2Cypher 生成查询语句；
-   查询校验保证安全。

流程：

``` text
用户问题
 ↓
需求理解
 ↓
生成查询
 ↓
安全检查
 ↓
图数据库分析
```

------------------------------------------------------------------------

## 向量检索辅助

知识图谱适合处理明确关系。

对于非结构化资料，结合：

-   FAISS；
-   Embedding。

用于：

-   相似项目匹配；
-   历史案例查询；
-   文档内容补充。

------------------------------------------------------------------------

## Agent流程设计

系统通过流程编排完成：

-   用户需求理解；
-   任务拆解；
-   工具调用；
-   查询执行；
-   结果整理。

根据不同业务进入对应处理流程。

------------------------------------------------------------------------

## 工程难点

### 文档结构复杂

需要处理不同格式招标文件，并完成内容标准化。

### 实体标准化

解决企业名称、证书名称、项目名称等不同表达问题。

### 查询安全

自然语言生成查询需要增加校验机制，避免错误查询。

------------------------------------------------------------------------

## 项目价值

该系统帮助企业将投标前审核流程由人工查找转变为智能辅助分析。

主要价值：

-   提升项目筛选效率；
-   降低审核成本；
-   提前发现资格风险；
-   辅助投标决策。
